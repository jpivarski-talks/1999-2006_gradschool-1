Info on How Cleog Runs Under Suez

The run number and number of events that suezCleog.exe uses is triggered externally to Cleog. Cleog is a producer and will produce what it's ordered to do. The run number is set by QQ, currently Cleog reads in a roar file from QQ but in the future QQ will run within Suez.


User Interface

Suez/Tcl is the "main" user interface/job control. Users can optionally use the FFREAD and KUIP user interfaces by setting optional Suez run-time parameters. These allow "experts" to specify additional commands. We want to continue to convert these switches to Suez run-time parameters so that we can completely remove these packages. We want all CLEO-III software to use a common framework and user interface. Note - when using KUIP you must enter 'quit' from the kuip prompt and not exit (a message is printed telling you to do this) as exit will exit the program - I didn't figure out how to disable this.


MCResponseProd's Suez Run-Time Parameters

How to change MCResponseProd's Suez Run-Time Parameters:
Suez> prod sel MCResponseProd
Suez> prod inter MCResponseProd
Suez> change parameter here
Suez> exit
  • UseKuip - Boolean to call KUIP at the begining of the job to access Cleog interactive commands. The default value is false.

  • KuipEveryEvent - Boolean to call KUIP before each event The default value is false. If you specify KuipEveryEvent you do not have to set UseKuip, if you do you'll call KUIP twice before the 1st event.

  • UseFFRead - Boolean to call FFREAD user interface package. The default value is false.
  • GeantBatch - Boolean to set Geant BATCH mode. Default is true. When BATCH mode is set, Geant doesn't prompt the user for the workstation type to open a higz window. If the user is running interactively and wants to open the higz window, set this parameter to false.
  • PrintGeantBanks Boolean to dump the Geant data structures at the begining of the job. The default is true.


KUIP

The kuip prompt is turned on by setting the Suez run time parameters UseKuip or KuipEveryEvent. See above.


FFREAD

Reading FFREAD cards is turned on by setting the Suez run-time parameter UseFFRead true.
